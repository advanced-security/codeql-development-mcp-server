# Integration Test: codeql_bqrs_interpret - graphtext_format

This integration test validates the `codeql_bqrs_interpret` MCP server tool with the `graphtext` output format.

## Test Description

Tests the basic functionality of interpreting BQRS (Binary Query Result Set) files into human-readable graphtext format, which is used for displaying graph query results such as AST (Abstract Syntax Tree) structures.

## Test Setup

### Before State

- `monitoring-state.json`: Empty sessions array (initial state)
- `results.bqrs`: Binary query results file to be interpreted

### After State

- `monitoring-state.json`: Contains session tracking information for the `codeql_bqrs_interpret` tool call
- `results.bqrs`: Same BQRS file (input files are not modified)
- `output.txt`: Expected output file containing the graphtext interpretation of the BQRS results (Note: This would be generated by the actual tool execution)

## Tool Parameters

```json
{
  "file": "/test/results.bqrs",
  "format": "graphtext",
  "output": "/test/output.txt",
  "t": ["kind=graph", "id=java/test-query"]
}
```

## Expected Behavior

The tool should:

1. Read the binary BQRS file
2. Interpret it according to the provided metadata (kind=graph, id=java/test-query)
3. Generate a human-readable graphtext output file
4. Track the operation in the monitoring state with success status

## Notes

- The `graphtext` format is specifically designed for `@kind graph` queries
- The `t` parameter provides required query metadata (kind and id) for interpretation
- This test validates the tool registration and basic parameter handling
